# 阶段实施 Skill

## 目标

严格实现 Web AI Coding Agent 项目的一个 milestone 阶段。

这个项目是教学型 Web AI Coding Agent。核心目标不是复制 Claude Code、Codex、OpenCode 的完整商业功能，而是理解并实现 coding agent 内核：agent loop、工具系统、上下文工程、patch 编辑、权限策略、验证闭环、trace 和 eval。Web UI 是产品展示入口，不应该承载 agent 核心逻辑。

## 开始前必须阅读

修改文件前，必须先阅读：

1. `AGENTS.md`
2. `ROADMAP.md`
3. `docs/PRODUCT_SPEC.md`
4. `docs/ARCHITECTURE.md`
5. 当前阶段文件：`docs/milestones/` 下对应的 phase 文件

如果这些文件缺失，不要猜测阶段范围，直接停止并报告缺失文件。

## 阶段信息提取

从当前 phase 文件中提取：

1. 阶段目标。
2. 允许实现范围。
3. 禁止实现范围。
4. 必须创建或修改的文件。
5. 必须更新的文档。
6. 验证命令。
7. 验收标准。
8. 最终报告格式。

这些内容不清楚时，不要开始实现。

## 标准工作流

1. 用一小段话复述当前阶段目标。
2. 列出计划创建或修改的文件。
3. 明确当前阶段禁止做什么。
4. 只实现当前阶段内容。
5. 保持 patch 小而可审查。
6. 更新对应设计文档。
7. 更新对应中文教程章节。
8. 更新对应中文博客草稿。
9. 运行 phase 文件中的验证命令。
10. 如果验证失败，使用 `validation-fixer` 的流程，最多重试 3 次。
11. 最终报告前使用 `code-reviewer` 流程自查。
12. 输出最终报告。

## 硬性规则

- 不要实现未来阶段。
- 未到对应阶段前，不要实现 VS Code 插件、MCP、云沙箱、subagents、eval。
- 不要擅自新增生产依赖；确实需要时，必须说明原因并请求确认。
- 不要重写无关模块。
- 不要掩盖验证失败。
- 没有实际运行验证命令时，不要声称验证通过。
- `agent-core` 必须独立于 `apps/web`。
- Web UI 保持轻量，agent 逻辑应放在 packages 中。
- 共享类型和协议放在 `packages/shared`。
- 文件写入、命令执行能力一旦出现，必须经过权限策略。
- 优先小修改，不要大重构。

## 文档要求

每个实现阶段必须创建或更新：

1. `docs/design/` 下的设计文档。
2. `docs/tutorial/` 下的中文教程章节。
3. `docs/blog/` 下的中文博客草稿。

文档必须和真实实现一致。没有实现的功能不能写成已经完成。

## 最终报告格式

输出：

1. 阶段名称。
2. 创建的文件。
3. 修改的文件。
4. 实现了什么。
5. 更新了哪些文档。
6. 运行了哪些命令。
7. 验证结果。
8. 是否遵守阶段边界。
9. 剩余限制。
10. 推荐的下一阶段。

## 停止条件

遇到以下情况应停止并报告：

1. 必要治理文件缺失。
2. phase 文件没有验证命令。
3. 当前任务必须依赖被禁止的未来功能。
4. 需要新增依赖但没有获得确认。
5. 同一个验证错误连续修复 3 次仍失败。
