# 从 Agent Loop 到 IDE 插件式任务体验

教学阶段的按钮叫“运行 Agent Loop”，这适合学习内部机制，但不适合产品。用户期待的是：像在 IDE 插件里一样发送消息，然后看到助手一步步工作。

P5 把 Agent Run 包装成 Product Task。

## 聊天框里应该看到什么

真正有用的聊天记录不只是 assistant 文本，还包括：

- 模型调用状态；
- 工具调用；
- 命令执行；
- 审批等待；
- 错误和停止原因。

这些都是用户理解 Agent 行为的关键线索。

## 为什么 Task 在 Server 层

Web 只负责渲染消息，不能承担任务状态机。P5 把状态机放在 Server/telemetry 层，让刷新恢复、审计和后续 resume 都有统一来源。

## 效果

用户现在可以在网页端获得接近 Codex IDE 插件的体验：右侧是聊天，工具和命令以消息形式出现，Diff Review 折叠在需要时查看。

## 还差什么

当前 resume 是状态恢复，不是后台进程恢复。后续还需要更完整的任务队列、可编辑 planner、多文件 apply 和 Git 输出。
