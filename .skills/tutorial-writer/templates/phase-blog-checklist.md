# 阶段博客检查清单

写完博客和教程后，必须逐项检查。

## 内容定位

- [ ] 这是一篇知识教学博客，不是技术设计文档。
- [ ] 文章围绕一个核心概念展开。
- [ ] 开头用具体问题引出概念。
- [ ] 概念解释在代码之前。

## 图示和图片

- [ ] 至少包含一张概念图。
- [ ] 至少包含一张流程图 / 架构图 / 时序图 / 状态机图。
- [ ] 每张图后都有文字解释。
- [ ] 图示用于解释知识点，而不是装饰。
- [ ] 精确架构没有使用 AI 生成图替代 Mermaid/SVG。
- [ ] 如使用截图，截图来自真实运行效果。
- [ ] 没有伪造截图或图片结果。
- [ ] 图片资源路径清晰，位于 `docs/blog/assets/phase-XX/`。

## 代码实现

- [ ] 引用了当前阶段真实代码路径。
- [ ] 代码片段简短，围绕知识点。
- [ ] 每段代码后都有解释。
- [ ] 说明了代码属于 Web / Runtime / Workspace 哪一层。
- [ ] 没有粘贴过长无关代码。

## 效果展示

- [ ] 包含 Web UI、API、终端输出、Trace、Diff 或验证结果中的至少一种。
- [ ] 命令输出来自真实运行。
- [ ] 没有声称未验证内容已经通过。
- [ ] 如果没有效果截图，明确写了 TODO 或原因。

## 局限性

- [ ] 明确说明当前阶段没有实现什么。
- [ ] 没有把未来阶段能力写成当前已完成。
- [ ] 解释了为什么本阶段只做到这里。

## 教学质量

- [ ] 语言清晰，适合学习者。
- [ ] 没有空泛宣传。
- [ ] 没有堆砌术语。
- [ ] 每个术语都有解释或上下文。
- [ ] 结尾连接到下一阶段。
